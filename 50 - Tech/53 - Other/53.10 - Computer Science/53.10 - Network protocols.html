  • A protocol is a set of rules that instruct how to do something
    • All devices can communicate with one another because they use protocols
  • Protocols determine:
    • How a device is identified
    • How communication should start and end
    • Which network port communication should use
    • What route data will take
    • How errors will be checked
    • How to acknowledge if data is received
    • What to do if data isn't received
    • How data is formatted and sequenced
    • How the speed of the sender and the receiver can be synced
  • Email protocols:
    • There are several email protocols
      • SMTP:
        • SMTP stands for Simple Mail Transfer Protocol, and it allows users to send emails
      • POP:
        • POP stands for Post Office Protocol, and it allows emails to be downloaded
        • The server stores the email until you request it, after which the email is on your device and the server deletes it
        • The advantage of this is that you can read your emails while not connected to the internet
        • The disadvantage is that you can't view your emails from another device
      • IMAP:
        • IMAP stands for Internet Message Access Protocol
        • It allows you to download emails from a server, however the server still keeps a copy of the email on the server
        • IMAP syncs both the client and the server to have the same emails
        • The advantage of this is that you can access your emails from any device
        • The disadvantage is that this can take up a lot of space on server
  • Wi-Fi:
    • Wi-Fi is a set of standards that determined how network devices should communicate wirelessly, using radio waves
    • The official name of Wi-Fi is IEEE 802.11x
    • It defines rules such as:
      • What frequency radio waves should use
      • How powerful the transmitter should be
      • How data should be secured
      • What happens if data is lost
    • There are different standards that operate at different speeds:
      • 802.11g (54Mb/s)
      • 802.11n (65Mb/s-600Mb/s)
      • 802.11ac (78Mb/s-3.2Gb/s)
    • Wi-Fi transmits in two different frequencies:
      • 2.4Ghz
      • 5Ghz
    • Wi-Fi encryption
      • Wi-Fi encryptions can be intercepted
      • Security measures have been invented to prevent this
        • WEP:
          • WEP is very basic
          • It requires a password
            • This password can be cracked easily
          • None of the data is encrypted
        • WPA:
          • WPA is ok
          • It uses a password
          • It uses key-based authentication to decode data
          • WPA has already been broken, however it is more difficult to do
        • WPA2
          • WPA2 is the best at the moment
          • It uses better encryption than WPA to scramble data
  • Data packets:
    • Packets are small amounts of data sent through a network
    • Packets include:
      • The senders IP address
      • The recipients IP address
      • The type of packet
      • The amount of packets sent
      • The data being sent
      • A flag to indicate if the data arrived
      • A sequence number to reassemble the data
  • TCP/IP:
    • TCP/IP (Transmission Control Protocol / Internet Protocol) is a networking stack that describes how data should be formatted
    • It is a model that define four layers for networking
      • Application layer:
        • This is where network software operates
        • HTTP:
          • This defines how data should be exchanged between web browsers and web servers
        • FTP:
          • This defines how data should be transmitted between a server and a client
        • SMTP:
          • This defines how email should be sent
      • Transport layer:
        • This is where transporting data occurs
        • TCP:
          • Transmission Control Protocol is a protocol that determines how devices should connect
          • The destination device must confirm that the packet arrived
          • This makes the system reliable
      • Internet layer:
        • This is where devices are individually identified
        • IP:
          • Internet Protocol is a protocol that identifies individual devices
          • This information helps routers know where data should be sent
      • Data link layer
        • This is how packets are transmitted
        • Ethernet:
          • Defines how data should be transmitted physically
          • Wraps the packet in an Ethernet frame which includes a MAC address
            • This identifies the device the packet should be sent to
          • The packets are turned into electrical signals, which are then sent to the LAN's router
          • If the destination IP is on the LAN, the packet is sent there
            • If not, the packet is sent onto the internet
    • Each layer is a group of network protocols that work at a similar level
  • IP:
    • IP addresses are assigned to devices when they join the network
      • They are linked to the operating system of the device
      • The address can be configured
    • IP addresses can be static (permanent), or dynamic (changing)
    • There are two types of IP addresses:
      • IPv4 is the most common
        • It is made up of a 32 bit number, separated by dots, like so: 192.168.0.125
        • IPv4 addresses have started to run out
      • IPv6 is less common, but it has many more addresses
        • It is made up of a 64 bit number, separated by colons, like so: FE80:0000:0000:0000:0202:B3FF:FE1E:3462
  • MAC:
    • MAC stands for Media Access Control
    • MAC addresses are assigned to physical components
      • They are more difficult to change than IP addresses
    • MAC addresses are used to direct packages from one router to another
    • Multiple network cards in a computer have different MAC addresses
    • MAC addresses are made up of 48 bit numbers separated by colons, like so: 00:25:96:FF:FE:12
  • Error detection:
    • Packets state which number they are in the sequence, and how many there are
    • If all packets have been received correctly, a confirmation message is sent to the sender
    • Packets have checksums. If the checksums don't match when arriving, the packet is sent again
