  • Network topologies are different connections between devices on a network
  • Bus topology:
    • A single cable to which every device is connected
    • Messages are sent along the cable
    • At the end there is a terminator to prevent signals from bouncing back
    • Only one computer can transmit a packet at a time
Advantages Disadvantages
Cheaper More collisions
Easy to install Main cable damaged, whole network fails
All workstations see the others
Difficult to identify where fault is
  • Ring topology:
    • All devices are connected to the next device in a loop
    • The are no collisions
Advantages Disadvantages
Adding more devices doesn't affect traffic Main cable damaged, whole network fails
Easy to install Difficult to identify where the fault is
New devices requires network to be shut down
More expensive than bus
  • Star topology
    • All devices are connected to a central hub
    • The hub directs messages between devices
Advantages Disadvantages
A damaged cable won't prevent other devices from functioning If the hub fails, the network fails
If a switch is used the network is more efficient Expensive
East to locate faults
New devices can be added with no downtime
  • Mesh topology
    • All devices are connected to at least one other device, sometimes all of them
Advantages Disadvantage
Fault tolerant Expensive and complex
High performance
Nodes extend range
Data can be transmitted simultaneously
